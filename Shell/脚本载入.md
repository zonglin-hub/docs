要让这个脚本在系统中生效，你可以按照以下步骤操作：

---

### 1. 保存脚本
将优化后的脚本内容保存到一个文件中，例如命名为 `preview`（无扩展名）或
`preview.sh`。  
假设你保存在家目录下的 `bin` 文件夹：
```bash
mkdir -p ~/bin
nano ~/bin/preview
```
粘贴脚本内容，保存退出（`Ctrl+O`，`Ctrl+X`）。

---

### 2. 添加执行权限
```bash
chmod +x ~/bin/preview
```

---

### 3. 确保 `~/bin` 目录在 PATH 中
通常 Ubuntu 会自动将 `~/bin` 加入 PATH，如果未生效，可手动添加：
```bash
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

或者将脚本复制到一个已在 PATH 中的系统目录（如 `/usr/local/bin`，需要 sudo）：
```bash
sudo cp ~/bin/preview /usr/local/bin/
```

---

### 4. 安装依赖
确保已安装所有必需工具（按照之前的安装命令）：
```bash
# 安装 chafa、ffmpegthumbnailer
sudo apt update && sudo apt install chafa ffmpegthumbnailer

# 安装 glow（任选一种方式）
sudo snap install glow
# 或使用官方仓库安装

# 安装 bat（使用 cargo）
cargo install bat
# 并确保 ~/.cargo/bin 在 PATH 中
```

---

### 5. 使用脚本
现在你可以直接在终端中调用 `preview` 命令：
```bash
# 预览单个文件
preview 图片.jpg
preview 文档.md
preview 视频.mp4
preview 代码.py

# 预览多个文件（仅图片或视频会触发缩略图网格）
preview 图片1.jpg 图片2.jpg 图片3.png
preview 视频1.mp4 视频2.mkv
```

---

### 6. 注意事项
- **终端需支持 Sixel 图像协议**：  
  默认的 GNOME Terminal **不支持** Sixel，因此图片和视频缩略图可能无法显示。  
  解决方法：  
  - 使用支持 Sixel 的终端，如 `mlterm`、`xterm -ti vt340` 或配置后的 `foot`。  
  - 修改脚本中 `chafa` 的 `--format` 参数为其他兼容格式，例如：  
    - `--format kitty`（适用于 Kitty 终端）  
    - `--format iterm2`（适用于 iTerm2）  
    - 如果只想显示文件信息而非图片，可移除图片分支。

- **多个文件的类型混合**：  
  脚本仅检查第一个文件的 MIME 类型，如果混合了不同类型（如图片+文本），可能会按
  第一个文件类型处理。建议单独预览不同类型。

- **性能**：预览大量图片/视频时，`chafa` 和 `ffmpegthumbnailer` 会占用一定资源。

---

### 7. （可选）设置别名或缩短命令
如果你希望命令更短，例如用 `pv` 代替 `preview`：
```bash
alias pv='preview'
# 将这一行添加到 ~/.bashrc 使其永久生效
```

### Ubuntu 默认的 PATH 扩展逻辑

在 Ubuntu 用户的 **`~/.profile`** 文件中（这是一个登录时自动执行的脚本），通常会
包含类似下面的代码段：

```bash
# set PATH so it includes user's private bin if it exists
if [ -d "$HOME/bin" ] ; then
    PATH="$HOME/bin:$PATH"
fi
```

- 当用户登录系统（无论是图形界面登录，还是通过 `ssh` 或 `su -` 登录）
  时，`~/.profile` 会被自动执行。
- 该代码会检查用户主目录下是否存在 `bin` 文件夹，如果存在，则将其添加到 `PATH`环
  境变量的最前面。
- 因此，**你只需要创建 `~/bin` 目录，并把脚本放进去，下次登录时 PATH 就会自动包
  含该目录**。

---

### 如何利用系统自带机制

1. **创建 `~/bin` 目录**（如果尚未存在）：
   ```bash
   mkdir -p ~/bin
   ```

2. **将脚本放入该目录**：
   ```bash
   cp /path/to/your/preview ~/bin/
   chmod +x ~/bin/preview
   ```

3. **使 PATH 立即生效**（两种方式）：
   - **重新登录**（注销并重新进入桌面，或重新打开终端并执行 `login` shell）
   - **手动执行 `~/.profile`**（只在当前 shell 生效）：
     ```bash
     . ~/.profile
     ```

4. **验证**：
   ```bash
   echo $PATH    # 应看到 ~/bin 出现在最前面
   which preview # 应显示 ~/bin/preview
   ```

---

### 注意事项

- 该机制依赖于 **登录 shell** 执行 `~/.profile`。在图形终端（如 GNOME Terminal）
  中默认打开的是交互式非登录 shell，它不会自动读取 `~/.profile`，但会继承登录
  shell 的 PATH 环境变量。如果你在图形终端中刚创建 `~/bin` 并希望立即生效，可以
  使用上述 `. ~/.profile` 命令。
- 如果你使用的是 **Zsh**，类似逻辑可能出现在 `~/.zprofile` 或 `~/.zshrc` 中，但
  Ubuntu 默认安装的是 Bash，以上说明适用于 Bash。

这种方法完全利用系统预置的配置，无需手动修改任何文件，简洁且符合 Ubuntu 的设计习
惯。
