```rust
pub fn jump(nums: Vec<i32>) -> i32 {
    let mut step = 0;       // step变量用于记录跳跃的次数
    let mut nums_max = 0;   // nums_max变量用于记录当前能到达的最远位置
    let mut end = 0;        // end变量用于记录当前跳跃的边界
    // 遍历数组中的每个元素（除了最后一个元素）
    for (i, _) in nums.iter().enumerate().take(nums.len() - 1) {
        // 更新当前能到达的最远位置
        nums_max = nums_max.max(i + nums[i] as usize);
        // 如果当前索引等于跳跃的边界，说明需要增加一次跳跃
        if i == end {
            end = nums_max;  // 更新跳跃的边界为当前能到达的最远位置
            step += 1;       // 增加跳跃次数
        }
    }
    step  // 返回跳跃次数
}
```

函数的逻辑如下：
1. 初始化三个变量：`step`用于记录跳跃次数，`nums_max`用于记录当前能到达的最远位置，`end`用于记录当前跳跃的边界。
2. 遍历数组中的每个元素（除了最后一个元素），因为最后一个元素是目标，不需要再进行跳跃。
3. 在每次遍历中，计算当前索引加上该索引位置的值，可以得到从该位置起跳能到达的最远位置，并与`nums_max`比较，取最大值更新`nums_max`。
4. 如果当前索引等于`end`，说明已经到达了当前跳跃的边界，需要增加一次跳跃。此时，将`end`更新为`nums_max`，表示下一次跳跃的边界，并将`step`加一。
5. 遍历结束后，返回`step`，即达到数组最后一个元素所需的最小跳跃次数。
这个函数的时间复杂度是O(n)，其中n是数组的长度，因为它只需要遍历一次数组。空间复杂度是O(1)，因为它只使用了常数个额外空间。
