
**直接使用 dpkg 过滤**

```bash
dpkg -l | grep '^rc'     # 不会匹配 "arc-package"，只匹配状态为 rc 的包
```

在正则表达式中，`^` 表示**行的开始**。所以 `'^rc'` 匹配的是**以 "rc" 开头的行**。

根据输出显示，`vim-runtime` 包的状态是 **rc**，这表示：
- **r**：软件包已被移除
- **c**：配置文件仍然保留在系统中

要完全清理这个包（包括配置文件），请执行以下命令：

```bash
sudo dpkg --purge vim-runtime
```

或者使用 apt 命令：

```bash
sudo apt purge vim-runtime # 推荐，需要手动清理，可能会匹配失效，导致删除多余的包📦
```


这两个命令都可以删除包及其配置文件，但有以下关键差异：

| 特性         | `dpkg --purge` | `apt purge` |     |
| ---------- | -------------- | ----------- | --- |
| **依赖处理**   | ❌ 不处理依赖关系      | ✅ 自动处理依赖关系  |     |
| **智能清理**   | ❌ 只删除指定包       | ✅ 可能建议移除相关包 |     |
| **配置文件删除** | ✅ 删除所有配置       | ✅ 删除所有配置    |     |
| **推荐使用场景** | 低级操作、脚本        | 日常使用、用户操作   |     |


如果想一次性清理所有已卸载但保留配置文件的包，可以使用：
```bash
sudo apt autoremove --purge # 特殊包，匹配无法自动清理
```
