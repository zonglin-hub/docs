在Ubuntu 24.04中，即监控GPU使用情况，可以通过多种工具实现，具体方法取决于你的显卡品牌（NVIDIA/AMD/Intel）。以下是分步指南：

---

### 快速检查（通用方法）

__查看显卡型号：__

```bash
lspci | grep -i vga
```

这会列出你的显卡型号（如NVIDIA、AMD Radeon、Intel核显）。

__参考信息：__

```bash
zonglin@localhost:~$ lspci | grep -i vga
03:00.0 VGA compatible controller: Advanced Micro Devices, Inc. [AMD/ATI] Cezanne [Radeon Vega Series / Radeon Vega Mobile Series] (rev c6)
```

根据你的终端输出，你的显卡是 __AMD Radeon Vega 系列（移动版）__，属于 AMD 集成显卡/核显。在 Ubuntu 24.04 中，可以通过以下方法监控 GPU 使用率：

---

### 使用 `radeontop`（实时监控 GPU 使用率）

这是 AMD 显卡的专用工具，能直观显示 GPU 核心、显存、频率等实时状态。

__安装 `radeontop`__

```bash
sudo apt update
sudo apt install radeontop
```

__运行监控（需 root 权限）__

```bash
sudo radeontop
```

## __界面说明__

这些参数是AMD显卡性能监控工具（如`radeontop`）显示的实时数据，用于反映GPU的运行状态和资源使用情况。以下是这些参数的详细解释及其意义

---

### 1. __Graphics Pipe（图形渲染管线）__

- __含义__：图形渲染管线是GPU处理图形任务的核心部分，负责将三维模型转化为二维图像。
- __9.17%__：表示当前GPU的图形渲染管线占用率为9.17%，即正在处理的图形任务负载较低。

---

### 2. __Event Engine（事件引擎）__

- __含义__：事件引擎负责处理GPU内部的各种事件，如指令调度和任务分配。
- __0.00%__：表示当前事件引擎未处理任何任务，处于空闲状态。

---

### 3. __Vertex Grouper Tessellator（顶点分组器/曲面细分器）__

- __含义__：负责将三维模型的顶点分组，并对其进行曲面细分，以提高图形的细节表现。
- __0.00%__：表示当前没有进行曲面细分操作。

---

### 4. __Texture Addresser（纹理寻址器）__

- __含义__：负责处理纹理贴图的相关操作，如纹理的寻址和采样。
- __7.50%__：表示当前纹理寻址器的占用率为7.50%，可能正在处理与纹理相关的任务。

---

### 5. __Shader Export（着色器输出）__

- __含义__：负责将着色器处理后的数据输出到下一阶段。
- __7.50%__：表示着色器输出阶段占用率为7.50%，表明有部分数据正在被处理并输出。

---

### 6. __Sequencer Instruction Cache（指令缓存）__

- __含义__：用于缓存GPU指令，减少内存访问延迟。
- __0.00%__：表示当前指令缓存未使用。

---

### 7. __Shader Interpolator（着色器插值器）__

- __含义__：负责在顶点之间插值，计算像素着色器的输入值。
- __8.33%__：表示插值器占用率为8.33%，可能正在处理像素着色相关任务。

---

### 8. __Scan Converter（扫描转换器）__

- __含义__：将三维几何数据转换为屏幕上的二维像素。
- __8.33%__：表示当前占用率为8.33%，可能正在处理图形渲染任务。

---

### 9. __Primitive Assembly（图元组装器）__

- __含义__：负责将顶点数据组装成图元（如三角形）。
- __0.00%__：表示当前没有进行图元组装操作。

---

### 10. __Depth Block（深度模块）__

- __含义__：负责处理深度测试，确保图形渲染的正确性。
- __8.33%__：表示深度模块占用率为8.33%，表明正在进行深度测试。

---

### 11. __Color Block（颜色模块）__

- __含义__：负责处理颜色混合和输出。
- __7.50%__：表示颜色模块占用率为7.50%，表明正在进行颜色相关的渲染任务。

---

### 内存和时钟信息

1. __VRAM（显存）__：
   - __459M / 470M (97.70%)__：表示当前显存使用量为459MB，总容量为470MB，使用率高达97.70%，接近满载。

2. __GTT（图形内存管理表）__：
   - __566M / 7657M (7.40%)__：表示GTT当前使用566MB，总容量为7657MB，使用率较低。
3. __Memory Clock（显存时钟频率）__：
   - __1.59G / 1.60G (99.38%)__：表示当前显存时钟频率为1.59GHz，接近最大频率1.60GHz，使用率很高。
4. __Shader Clock（着色器时钟频率）__：
   - __0.60G / 1.80G (33.20%)__：表示当前着色器时钟频率为0.60GHz，远低于最大频率1.80GHz，使用率较低。

---

### 总结

这些参数反映了GPU各部分的运行状态和资源使用情况：

- __低负载__：如Event Engine、Vertex Grouper Tessellator等模块处于空闲状态。
- __中等负载__：如Texture Addresser、Shader Interpolator等模块占用率在7.50%-8.33%之间。
- __高负载__：如VRAM使用率接近满载，表明显存资源紧张。
通过这些数据，用户可以了解GPU的性能瓶颈，从而优化资源分配或排查潜在问题。如果需要进一步优化性能，建议关注显存使用情况和任务负载分布。
