以下是一套简洁实用的 Git Commit 规范指南，结合了主流的 Conventional Commits 规范，并为你提供了可操作性强的模板：

---

### **基础版规范（快速上手）**

**提交格式**：`<类型>: <主题>`

**1. 必填类型前缀**（常见类型）：

- ✨ `feat`: 新增功能/特性 (如：`feat: 新增用户导出功能`)
- 🐛 `fix`: 修复 bug (如：`fix(登录): 解决验证码不刷新问题`)
- 📝 `docs`: 文档更新 (如：`docs: 添加接口文档注释`)
- 🎨 `style`: 代码格式调整（不影响功能）(如：`style: 优化表格间距`)
- ♻️ `refactor`: 代码重构（非功能变更） (如：`refactor(订单): 拆分支付逻辑模块`)
- ⚡️ `perf`: 性能优化
- ✅ `test`: 测试相关
- � `chore`: 日常维护（依赖更新/构建配置等）(如：`chore: 更新 axios 到 v1.6`)

---

### **纯中文版方案（需工具适配）**

如果团队不使用自动化生成 CHANGELOG 的工具，可采用全中文：

```markdown
[新增][订单模块]: 支持导出物流单号
[修复]: 解决移动端弹窗滚动穿透问题
[样式]: 调整表格行高为 40px
```

### **全中文类型对照表**

| 类型标识 | 对应英文 | 适用场景说明                 |
|----------|----------|------------------------------|
| [新增]   | `feat`   | 添加新功能/特性               |
| [修复]   | `fix`    | 解决缺陷/错误                 |
| [文档]   | `docs`   | 文档相关变更                  |
| [样式]   | `style`  | 界面样式/代码格式调整          |
| [重构]   | `refactor` | 代码重构（不改变功能）        |
| [性能]   | `perf`   | 性能优化                      |
| [测试]   | `test`   | 测试用例相关                  |
| [维护]   | `chore`  | 日常维护/配置调整              |

### **模板使用规范**

1. **格式层级**  

   ```markdown
   [类型][模块]: 主题描述
   （空行）
   正文说明（可选）
   （空行）
   脚注标记（可选）
   ```

2. **写作建议**：
   - **模块标注**：当影响范围明确时使用（如 `[新增][订单模块]`）
   - **主题规范**：保持口语化但专业，如：

     ```markdown
     ✅ 正确：[修复][缓存]: 解决数据重复过期问题
     ❌ 避免：[修复][缓存]: 修了一个缓存bug
     ```

3. **复杂场景**：

   ```markdown
   [性能][图片加载]: 实现渐进式加载方案
   
   技术细节：
   - 优先加载500px缩略图
   - WebP兼容性检测
   - 平均加载时间减少62%
   
   性能报告：/docs/perf-2024Q2
   ```
