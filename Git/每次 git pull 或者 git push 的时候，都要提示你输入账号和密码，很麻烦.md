每次 git pull 或者 git push 的时候，都要提示你输入账号和密码，很麻烦。
于是在网上找到了解决办法。这j里记录下来，以便以后解决此类问题。

首先进入你的 git 工作目录下。
在终端执行一下命令：

```bash
zonglin@localhost:~$ # 配置 Git 记住你的用户名和密码
zonglin@localhost:~$ git config --global credential.helper store
```

具体解释如下：

1. **`git config`**: 这是 Git 的命令，用于设置和查看 Git 的配置信息。
2. **`--global`**: 这个选项表示这个配置是 **全局生效** 的。也就是说，这个设置会应用到你的操作系统当前用户账户下的 **所有 Git 仓库**，而不仅仅是当前目录下的某个特定仓库。配置文件通常位于你的用户主目录下 `~/.gitconfig`。
3. **`credential.helper`**: 这是你要设置的配置项的名称。它告诉 Git 使用哪个 **凭证助手** 来处理认证信息（如访问远程仓库时需要的用户名和密码）。
4. **`store`**: 这是你为 `credential.helper` 指定的值。`store` 是一个内置的、简单的凭证助手。它的作用是：
    * **保存凭证**：当你第一次通过 HTTPS 协议向远程仓库（如 GitHub、GitLab 等）推送代码或拉取需要认证的代码时（例如 `git push`, `git pull`, `git fetch`, `git clone` 私有仓库），Git 会提示你输入用户名和密码。
    * **明文存储**：输入正确的用户名和密码后，`store` 助手会**以明文形式**将它们保存到你**用户主目录下的一个文件**中 `~/.git-credentials`。
    * **自动填充**：之后，当你再次执行需要认证的 Git 操作时，Git 会自动从这个文件中读取你的用户名和密码，**不再需要你手动输入**。

**总结来说，执行这个命令的意思是：**

> “告诉 Git，为当前系统用户的所有仓库，启用 `store` 凭证助手。这样 Git 就能把我第一次输入的远程仓库用户名和密码（明文）保存到主目录的一个文件里，以后访问同一个远程仓库时就自动使用这些信息登录，不用我再每次都输入了。”

**重要注意事项：**

* **明文存储风险**：`store` 方式最大的缺点是**密码以明文形式存储**在你的电脑上。如果其他人获得了对你用户账户或该文件的访问权限，他们就能看到你的密码。**请确保你的操作系统账户安全！**
* **适用范围**：主要用于 **HTTPS 协议** 克隆/访问的仓库。SSH 协议使用密钥认证，不受此配置影响。

如果你主要是为了方便且对个人电脑的安全性有信心，`store` 是一个简单直接的选择。如果处理敏感仓库或使用共享电脑，**强烈建议使用更安全的替代方案（如 `manager-core`, `osxkeychain` 或 `cache`）。**

然后再重新执行一次git pull,这次它还是提示你输入账号和密码：

示例：

```bash
zonglin@localhost:~$ git pull
Username for 'https://isource-sh.dajiahao.com': lzlysfxx@163.com
Password for 'https://stxgs123@isource-sh.dajiahao.com':*******
```

这次之后，它就会记录下账号和密码。以后再 pull 或者 push 的时候，
再也不用输入账户和密码了，节省时间，提高效率！！
