国内使用 `winget` 卡顿的主要原因是默认源（微软官方源）在国内访问速度较慢。目前可通过更换国内镜像源或优化配置来提升体验。以下是可用的解决方案及相关镜像站信息：

### 1. __中科大开源软件镜像站__

- __镜像地址__：`https://mirrors.ustc.edu.cn/winget-source`
- __配置步骤__：
  1. 以管理员身份打开命令提示符。
  2. 移除默认源：

     ```bash
     winget source remove winget
     ```

  3. 添加中科大镜像源：

     ```bash
     winget source add winget https://mirrors.ustc.edu.cn/winget-source
     ```

  4. 验证配置：

     ```bash
     winget source list
     ```

- __特点__：镜像源同步了开源软件包，但微软官方商店的应用程序可能仍需通过官方源下载。目前中科大是国内唯一公开的 `winget` 镜像源。

### 2. __其他加速建议__

- __使用代理或网络加速工具__：若镜像源仍不理想，可尝试通过代理工具（如 Clash、V2Ray 等）提升国际网络访问速度。
- __检查网络配置__：部分网络问题可能由 DNS 解析或 IPv4/IPv6 冲突导致，可尝试优化 `/etc/resolv.conf` 或禁用 IPv6。
- __多线程下载工具__：对于 `winget` 无法直接优化的场景，可尝试结合 `aria2` 等第三方工具下载安装包（需手动操作）。

### 3. __注意事项__

- __镜像源兼容性__：中科大镜像源主要覆盖开源软件包，部分微软商店应用可能仍需切换回官方源下载。
- __恢复默认源__：若需恢复微软官方源，执行以下命令：

  ```bash
  winget source reset winget
  ```

### 总结

目前国内可用的 `winget` 镜像站主要为中科大开源镜像站，其他镜像源尚未广泛支持。若仍存在速度问题，建议结合代理工具或优化本地网络环境。更多操作细节可参考[中科大镜像站配置指南](https://www.sysgeek.cn/windows-winget/)。
